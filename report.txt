Experiment Suite 1

General Setup

All experiments were run on randomly generated directed weighted graphs with edge weights between 1 and 20. The source node was fixed at node 0. Each experiment was repeated 12 times and the average result was recorded.

Approximation quality was measured using relative error:
(approx_total - exact_total) / exact_total

Only finite distances were included in the total. Runtime was measured using time.perf_counter().


Experiment 1: Effect of k on Approximation Quality

Design:
The graph size was fixed at n = 50 and complete graphs were used. The value of k was varied over {1, 2, 3, 5, 10, 20}.

Reason:
Complete graphs contain many possible paths, which makes it easier to observe the impact of limiting relaxations. The chosen k values cover both small and large limits.

Results:
Figure 1

Discussion:
The results show that k has a strong impact on accuracy. When k is small, many useful relaxations are missed, which leads to large errors. As k increases, the approximation improves quickly.

Around k = 5, the error becomes very small. By k = 10, both algorithms reached zero relative error in our trials.

Dijkstra Approx improves faster than Bellman-Ford Approx. This is likely because Dijkstra uses a priority queue, which allows it to focus on shorter paths earlier.


Experiment 2: Effect of Graph Size on Runtime

Design:
The value of k was fixed at 5 and complete graphs were used. The number of nodes was varied over {10, 20, 50, 100}.

Reason:
These values provide a range of graph sizes while keeping the experiment practical. Fixing k allows the effect of graph size to be isolated.

Results:
Figure 2

Discussion:
Runtime increases as the graph size increases. Larger graphs contain more edges, so both algorithms need to process more data.

Dijkstra Approx is consistently faster than Bellman-Ford Approx, and the difference becomes more noticeable as n increases. This matches the expected behavior based on their algorithm design.


Experiment 3: Effect of Graph Density on Approximation Quality

Design:
The graph size was fixed at n = 50 and k was fixed at 3. Graph density was varied over {0.1, 0.3, 0.6, 1.0}.

Reason:
Fixing n and k allows the effect of density to be studied directly. The selected values range from sparse to fully connected graphs.

Results:
Figure 3

Discussion:
The results show an overall trend that approximation quality becomes worse as graph density increases.

With a fixed k, the algorithm cannot explore all possible improvements in dense graphs. As a result, it is more likely to miss better paths.

Dijkstra Approx remains more accurate than Bellman-Ford Approx across all densities.


Overall Conclusion

The experiments show that k is the main factor controlling approximation quality. Larger k values reduce error and can eventually recover the exact shortest paths.

Graph size mainly affects runtime, with both algorithms becoming slower as the graph grows.

Graph density also affects approximation quality. When k is small, denser graphs can lead to worse approximations because there are more possible paths than the algorithm can explore.
